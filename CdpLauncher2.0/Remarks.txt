2.0.1.0
	- ускорена загрузка файлов и разбор его по продуктам
	- возможность использовать часть имени файлов в качестве признака продукта
	- допобработка строк после cdp, использование секций внутри элемента массивов (и текст и бинарный)

2.0.2.0
	- допобработка строк перед cdp, добавление данных (справа, слева)
	- использование входных данных в качестве части имени выходного файла (разбивка по дизайнам). Параметр AdvanceUse
	- можно запускать без обработки cdp, тогда просто по сути разбивка по продуктам. Параметр Cdp="none" в продукте

2.0.3.0
	- параметр NoPin=true и NoProc=true в продуктах. Даже если cdp выгружается эти файлы, то можно отказываться от их переноса
	- добавлен тип условия ItemOfCSV (то есть можно что-то указывать через запятую)

2.0.3.1 (06.06.2019)
	- атрибут Length для допобработки строк перед cdp
	- атрибут EncodingOut тэгов OutPinFileFormat и OutProcFileFormat

2.0.3.2 (14.06.2019)
	- атрибут PostAction секции Input - что делать с входным файлом после того как все его карты обработаны. (None, Delete, Move)
		если Move, то берется тэг ArchiveDirectory
	- теперь когда отрабатывает функция LoadData все файлы оказываются выбранными (чтобы можно было сразу жать кнопку запуска)
	- атрибут продукта Ignore, если true, то вообще этот продукт никак не обрабатывать, сразу считаем обработанным (например, для строк DCL)

2.0.3.3 (27.06.2019)
	- атрибут Encoding тэга OutFileFormat (может быть в секции CDP или уже в продукте)
	- атрибут Function="ArrayItem" для Item в OutFileName (еще атрибуты Separator, Index)

2.0.3.4 (02.08.2019) 
	- тип постобработки ReformName - более гибко можно использовать переформирование имен (по сравнению с RegenName).
	- для этого в OutFileNameFormat добавлен тип InputField
	- в постобработки введены постобработка для файлов пинов и процессинга (AddOnPin, AddOnProc)
	- язык программы выставляет в основном xml

2.0.4.0 (19.08.2019) (последние два пункта сделаны 22.08 - обе версии ушли в розан спб)
	- тип постобработки RemoveRecord - удалить запись по какому то признаку из выходного файла (атрибуты Delimitor, Count, StartPos, Length, Value)
		сделано в первую очередь для исключения из итогового файла по пинам тех записей, у которых есть признак отсутствия пин-конверта
	- атрибут Ignore в products.xml, не только не обрабатывается при запуске, но и не отображает эти продукты в окне
	- для поддириректорий итоговых файлов добавлено Type="ProductName" и поддержка функций substring, arrayitem (было только Const)
	- исправлен баг отображения ошибок (при переходе с string.format на $ неверно обрабатывался {0})
